Personal Project

Self-Directed · Vibe-Coded · Open Source 🐳 Fully Dockerized · 7 Containers

TestFlow — Test Case Management Platform

A full-stack test case management platform covering the complete test lifecycle — Projects → Suites → Test Cases → Runs → Results — with AI-powered test case generation and real-time run collaboration. Built as a vibe-coding showcase for how AI-assisted development can rapidly produce production-like engineering: it ships in two interchangeable deployment modes behind the same API surface — a FastAPI monolith (deployed on Vercel) and a 5-service microservice architecture (Docker Compose + Postgres + Redis).

🐳
Docker end-to-end. Every deployment mode ships as a container: a standalone Dockerfile (Python 3.11-slim + uvicorn) for the monolith, a 2-container docker-compose.yml (app + Postgres, with a health-checked DB dependency) for local/monolith use, and a 7-container docker-compose.microservices.yml — gateway, auth, projects, runs, ai, postgres:16-alpine, redis:7-alpine — for the full microservice topology, all brought up with one command.

Data flow & architecture (microservice mode):

🌐 Browser — Vanilla JS SPA Chart.js analytics dashboard
HTTP / REST + WebSocket
🐳 Docker Compose — docker-compose.microservices.yml
🔐 Gateway :8000 httpx proxy · static file serving 🐳 container
🔑 Auth :8001 JWT login · users 🐳 container
📁 Projects :8002 CRUD · stats · analytics 🐳 container
▶️ Runs :8003 Runs · results · WebSocket 🐳 container
🤖 AI :8004 Claude Haiku test generation 🐳 container
🐘 PostgreSQL 16 schema: auth (users)
schema: projects (projects, suites, test_cases)
schema: runs (test_runs, results) 🐳 postgres:16-alpine
🔴 Redis 7 channel: runs.completed
(async event pub/sub) 🐳 redis:7-alpine

Every box inside the dashed boundary is its own Docker container, brought up together with a single docker compose -f docker-compose.microservices.yml up --build — only the Gateway container needs to be exposed publicly. JWT carries a role claim so downstream services verify tokens locally instead of round-tripping to Auth on every request; Redis pub/sub delivers runs.completed events fire-and-forget and degrades gracefully if Redis is down.
